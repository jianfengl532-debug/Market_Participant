Alpine Linux 本机配置查阅指南

在 Alpine Linux 系统中，由于它主打精简和轻量化，默认没有安装像 neofetch 或 lshw 这样的重型复杂工具。不过，你可以通过 Alpine 自带的系统文件和轻量命令完整地查阅本机配置。

以下是查阅不同硬件和系统配置的常用命令及实际输出参考：

1. 查看系统版本与内核信息

# 查看 Alpine Linux 的具体版本号
cat /etc/alpine-release
3.23.4

# 查看当前运行的系统内核版本及架构
uname -a
Linux jefflu 6.18.33-0-lts #1-Alpine SMP PREEMPT_DYNAMIC 2026-05-24 18:06:34 x86_64 Linux

6. MariaDB 服务状态概览

项目信息
版本MariaDB 11.4.11
运行状态✅ 运行中（PID: 3314）
数据目录/var/lib/mysql
Socket/var/run/mysqld/mysqld.sock
端口3306（TCP, 绑定 0.0.0.0）
配置文件/etc/my.cnf, /etc/my.cnf.d/mariadb-server.cnf

配置要点：

  • skip_ssl — SSL 已禁用
  • bind-address = 0.0.0.0 — 允许远程连接
  • max_connections = 10
  • innodb_buffer_pool_size = 20M
  • skip_name_resolve = 1
⚠️ 注意：当前以用户 jefflu 运行，但 MariaDB 的 root 用户使用了 unix_socket 身份认证插件。这意味着只有 OS 的 root 用户才能以 MariaDB root 身份登录，当前用户没有权限直接查询数据库内容。 如需查看数据库，可尝试以下方式： 
# 方式一：以 root 系统用户执行
    su -c "mariadb -e 'SHOW DATABASES;'"

    # 方式二：提供密码远程连接（如果配置了密码认证用户）
    mariadb -h 127.0.0.1 -u root -p -e "SHOW DATABASES;"
