修改网页内容后 → 只需做这四步：

cd /share/www
git add 你改动的文件路径
git commit -m "改了什么东西"
git push

⚠️ 首次推送需配置认证方式（见第三节说明）。建议用 git add 精确指定文件，避免提交无关改动。

以上两步涵盖了日常所有操作。以下为详细说明。

一、本地开发环境启动

启动 PHP 后端（数据层 :8000）：

cd /home/jefflu && sh start.sh start

启动 Python 前端（展示层 :8080，本地开发用）：

python3 -m http.server 8080

二、生产环境启动（Funnel HTTPS）

1. 确保 PHP 后端在运行：

ps aux | grep php

2. 启动 Tailscale Funnel（持久化后台运行）：

nohup tailscale funnel 8000 > /dev/null 2>&1 &

3. 验证 Funnel 是否生效：

wget -q -O- --header="X-API-Key: mk_jfl_a8f3c9e2b7d4p6q5r1" "https://jefflu.tail9f69ce.ts.net/ping.php"

4. 前端由 GitHub Pages 提供，无需启动本地前端服务器：
https://jianfengl532-debug.github.io/Market_Participant/

三、修改网页后推送（每次改完都要做）

1. 进入前端仓库，查看改了什么文件：

cd /share/www
git status

2. 添加改动的文件（推荐精确指定，避免提交无关文件）：

git add Data_Processing_Analysis/Finance/PerAcc/PerAcc.html

如果改了多个文件，也可以用 git add . 添加全部。

3. 提交本次改动，附上说明：

git commit -m "PerAcc: 数据更新时间标注 + 列表姓名脱敏"

4. 推送到 GitHub（不要加 -f 参数，避免覆盖远程记录）：

git push

推送时会提示输入 GitHub 用户名和密码（密码用 Personal Access Token）。

5. ⚠️ 如果遇到以下错误：

fatal: The current branch main has no upstream branch.
To push the current branch and set the remote as upstream, use:

    git push --set-upstream origin main

说明当前分支没有设置上游跟踪分支（常见于刚克隆或新建分支后第一次推送），执行：

git push --set-upstream origin main

这条命令会推送并自动建立跟踪关系，以后只需 git push 即可。

6. 等 1-2 分钟后刷新 GitHub Pages 页面即可看到更新。

⚠️ 首次推送 / 认证配置

GitHub 已于 2021 年禁用密码认证，必须使用 Personal Access Token (PAT) 或 SSH。

1. 设置 Remote URL（如未配置过）：

git remote set-url origin https://jianfengl532-debug@github.com/jianfengl532-debug/Market_Participant.git

2. 生成 Personal Access Token：
GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
点「Generate new token (classic)」→ 勾选 repo 全部权限 → 生成并复制 token。

3. 推送时输入 token：
执行 git push 后提示输入密码，粘贴刚才的 token（不是 GitHub 登录密码）。
验证成功后会显示：

Enumerating objects: ...
remote: Resolving deltas: 100% (...), completed ...
To https://github.com/jianfengl532-debug/Market_Participant.git
   xxxxxx..xxxxxx  master -> master

免密推送（可选）：运行一次以下命令，之后不再询问密码：

git config --global credential.helper store

🔁 六、服务器重启完整恢复流程

服务器（Alpine Linux）重启后，所有服务都会停止。按以下顺序逐步恢复：

第1步：检查互联网连通

ping -c 2 8.8.8.8

有回复说明网络正常。如果 ping 不通，先检查网线/路由器。

第2步：启动 MariaDB 数据库

rc-service mariadb start

MariaDB 已配置开机自启，但偶尔不会随系统自动启动，需要手动确认：

rc-service mariadb status

显示 * status: started 即可。

第3步：启动 PHP 后端 + Python 前端

cd /home/jefflu && sh start.sh start

这会同时启动：

  • PHP 后端（数据录入页 + API）→ localhost:8000
  • Python 前端（本地开发用展示页）→ localhost:8080

第4步：开放外网访问（Tailscale Funnel）

nohup tailscale funnel 8000 > /dev/null 2>&1 &

这条命令让 GitHub Pages 能通过 Funnel 域名访问到后端的 PHP API。

第5步：验证全链路

# 检查数据库
rc-service mariadb status

# 检查本地 API 是否响应
curl -s --header "X-API-Key: mk_jfl_a8f3c9e2b7d4p6q5r1" http://localhost:8000/ping.php

# 检查外网 API 是否响应
curl -s --header "X-API-Key: mk_jfl_a8f3c9e2b7d4p6q5r1" https://jefflu.tail9f69ce.ts.net/ping.php

全部返回 {"status":"ok", "db":true} 说明链路通畅，GitHub Pages 前端即可正常显示数据。

第6步：推送未完成的 git 提交（如有）

cd /share/www
git push

如果提示 no upstream branch，执行：

git push --set-upstream origin main

👉 推送时提示输密码，输入 Personal Access Token (PAT)，不是 GitHub 登录密码。

📌 汇总命令（一条执行所有）

rc-service mariadb start && \
cd /home/jefflu && sh start.sh start && \
nohup tailscale funnel 8000 > /dev/null 2>&1 & && \
echo "✅ 重启恢复完成"
