## 一、改造目标

将单文件 `index参考.html` 拆分为三个独立文件，实现**结构（HTML）**、**样式（CSS）**、**逻辑（JS）** 的分离。

## 三、拆分方式

| 文件 | 来源 | 说明 |
|------|------|------|
| `index.html` | 原文件的 `<body>` 结构 + 引用替换 | 删掉 `<style>` 和 `<script>` 块，替换为外部引用 |
| `css/style.css` | 原文件 `<style>` 标签全部内容 | 按功能分区加注释，新增 `.footer` 类（替代原页脚内联样式） |
| `js/script.js` | 原文件 `<script>` 标签全部逻辑 | **重构了事件绑定，消除冗余**（详见下文） |

### HTML 引用方式

```html
<!-- 样式引用 -->
<link rel="stylesheet" href="css/style.css">

<!-- 脚本引用 -->
<script src="js/script.js"></script>
```

## 四、JS 重构要点（重要）

原参考文件在迭代过程中积累了 **5 个问题**，拆分时一并清理：

### 1. 函数覆盖问题
- **原文件**：`openModal` 先在第 465 行定义，又在第 509 行重新声明覆盖（为了修补赞赏按钮的显示恢复）
- **重构后**：拆分为两个独立函数 `showToolModal(toolId)` 和 `showDonateModal()`，互不干扰

### 2. 重复事件绑定
- **原文件**存在多处重复监听：
  - 下载按钮点击 → 绑定 2 次（第 480 行、第 533 行），点击一次触发两次
  - 关闭按钮点击 → 绑定 2 次（第 487 行、第 560 行）
  - 遮罩层关闭 → 绑定 2 次（第 488 行、第 564 行）
- **重构后**：每个事件只注册一次

### 3. 赞赏按钮的 hack 逻辑
- **原文件**：赞赏按钮手动隐藏弹窗中的链接区域（`display: none`），再由 `openModal` 恢复显示，两者互相覆盖
- **重构后**：`showDonateModal()` 和 `showToolModal()` 各自独立设置完整状态

### 4. 新增函数清单

```javascript
// 工具数据（可在此数组增删工具）
const tools = [ ... ];

// 渲染所有卡片
function renderCards()

// 打开工具下载弹窗
function showToolModal(toolId)

// 打开赞赏弹窗
function showDonateModal()

// 关闭弹窗
function closeModal()
```

### 5. 事件绑定方式

- 下载按钮使用**事件委托**（监听 `document`）：即使卡片动态渲染也能响应，无需重新绑定
- 赞赏按钮、关闭按钮、遮罩点击均为直接绑定
