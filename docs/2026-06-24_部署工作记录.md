## 二、安全层（Phase 1）

### 2.1 CORS 白名单

- **位置：** `/home/jefflu/api/db_config.php` → `setCorsHeaders()`
- **允许的域名：**
  - `https://jianfengl532-debug.github.io`
  - `http://localhost:8080`
  - `http://127.0.0.1:8080`
- **拒绝：** 未知 Origin 的浏览器请求

### 2.2 API Key 校验

- **后端密钥：** `/home/jefflu/api/db_config.php` 中 `define('API_KEY', '...')`
- **前端密钥：** `/share/www/js/config.js` 中 `apiKey`
- **校验方式：** 所有 API 请求必须在 Header 携带 `X-API-Key`
- **未授权响应：** `401 Unauthorized`

### 2.3 数据库安全

- 数据库监听 127.0.0.1:3306，仅本地 PHP 可连
- 所有查询使用 PDO prepared statement，防 SQL 注入

## 五、已知问题

1. **Funnel 稳定性** — 偶发 `Done(1)` 退出，需重新 `nohup` 启动
2. **Alpine Git push** — 偶发 `SSL_read: unexpected eof`，断网重连或关终端重开可解决
3. **记住密码** — 已配 `credential.helper=store`，后续 push 不应再问密码
4. **Push 长时无响应** — 可按 `Ctrl+C` 中断，重新 `git push` 重试
