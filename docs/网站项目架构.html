1. 项目概览

个人投资研究与数据分析平台，包含三个分析模块 + 三个数据模块（价格/个人账户/企业财务）+ 一个订单系统。

部署模式：前端静态文件托管于 GitHub Pages（HTTPS），后端 PHP API 通过 Tailscale Funnel 暴露为 HTTPS，数据留在本地笔记本。远程维护通过 Tailscale SSH。

模块一览：🌍 GeoEnv（地缘政治分析）、💱 FxCap（汇率利率分析）、🏭 IndBiz（产业分析）、📊 Price（价格数据/斐波那契计算）、💳 PerAcc（个人账户管理）、🏢 FinData（企业财务数据）。

6. 关键技术细节

项目说明
DB.js核心引擎（992 行），提供表单渲染、API 调用、localStorage 降级。GeoEnv / FxCap / IndBiz 三个模块共享，通过 DB.init('模块名') 切换。解析器已拆分到独立的 parsers.js（504 行）。
parsers.js独立的文本解析器文件，包含 GeoEnvParser、FxCapParser、IndBizParser、PriceParser 四个解析函数，解析黏贴文本为结构化 JSON。由 DB.js 在解析时调用。
API（分析模块）GeoEnv / FxCap / IndBiz 各自独立的 api.php，支持 list（GET）、save（POST）、update（POST）、delete（POST）。
API（Price 模块）Price_Data/api.php 额外支持 query（按代码查询）、calc_save（保存斐波那契结果）、calc_list（列出计算结果）、autocomplete（代码模糊搜索 + 自动填充）。
Price 代码唯一性save 接口采用 upsert 逻辑：同一代码已存在则更新价格，不存在则插入。数据库 price_spot_data.code 有 UNIQUE 约束 uk_price_code。前端 autocomplete 输入"cu"即弹出匹配代码供选择。
PerAcc 身份证唯一性save 和 update 接口在写入前检查 id_number 是否重复。数据库 per_accounts.id_number 有 UNIQUE 约束 uk_id_number。检测到重复时返回错误"身份证号已存在：XXX（记录 #N）"，前端显示错误不清空表单。
API（PerAcc 模块）PerAcc_Data/api.php 额外支持 search（姓名/身份证/手机号搜索）和 detail（单条完整记录含账户/持仓/出入金）。
PerAcc 前端展示页 Finance/PerAcc/PerAcc.html 使用 Finance.js 中的 PF 命名空间（调用 PF.init），数据录入页使用独立的 PerAcc_Data.js（793 行），不共享 DB.js。

API Key 安全层所有 API 请求必须在 Header 携带 X-API-Key，密钥定义在 db_config.php（后端）和 js/config.js（前端）。不匹配时返回 401。前端通过 config.js 统一注入，无需手动管理。
config.js 环境切换/share/www/js/config.js 检测 window.location.hostname：本地开发时使用 localhost:8000，生产时使用 Funnel URL https://jefflu.tail9f69ce.ts.net。所有显示页统一引用此文件。
展示页 DB 链接使用 window.location.hostname 动态解析 + js/config.js 配置覆盖。生产环境指向 Funnel URL：https://jefflu.tail9f69ce.ts.net。
健康检查ping.php 返回服务器状态和数据库连通性。本地访问 http://localhost:8000/ping.php，线上访问 https://jefflu.tail9f69ce.ts.net/ping.php（需带 API Key）。
启动脚本（本地开发）/home/jefflu/start.sh 一键启停前后端服务（Python http.server + PHP），支持 start|stop|status 子命令。
Funnel 启动（生产）后端通过 Tailscale Funnel 暴露为 HTTPS：tailscale funnel 8000。生产环境只需启动 PHP 后端，前端由 GitHub Pages 提供。
唯一性迁移脚本migrate_unique.sql 给已有数据库添加 UNIQUE 约束（代码 + 身份证），自动清理重复数据，不丢记录。
FinData reports 结构fin_enterprises.financial_data 以 JSON 存储多周期报告：{"2025-Q4":{period_label, financial_data}, "2026-Q1":{...}}。前端自动对比展示各周期财务数据。
FinData 银行流水（2026-06 新增）独立表 fin_bank_flows，通过 uscc 与企业关联。前端展示页同时加载 fin_enterprises 和 fin_bank_flows 数据，自动合并渲染。每条记录标注数据类型（财务报告/银行流水/两者皆有）。导入流程：前端 JSON → PHP 调用 Python → 生成现金流量表 HTML → 存入数据库。

FinData 社会统一代码唯一性fin_enterprises.uscc 有 UNIQUE 约束 uk_uscc。支持 save（新建企业）和 add_period（追加周期）两种保存模式。
FinData 周期管理录入时手动输入报告期间（格式 YYYY-QN，如 2025-Q4）。同一企业可分多次导入不同周期的财务数据，自动合并展示。
